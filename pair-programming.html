Pair Programming

What is pair programming?

  • collaboration
  • communication
  • educational
  • productive
  • effecient

Benefits

  • seasoned pairs describe working "more than twice as fast"
  • design quality improves
  • fewer bugs(because they're caught earlier)
  • shared knowledge
  • faster problem solving

Source:"The Costs and Benefits of Pair Programming"

How to Pair

  • find somone to code with
  • one person is the driver
    • in control of inputting data
  • one person is the "navigator"
    • reponsible for reviewing code
    • refers to documentation

Getting started

  1. Define a task
  2. Focus on one task at a time
  3. Share your thought process with your pair

Switch roles

  • Try each role for 30 minutes at a time
  • Work with others

Evaluate

  • Spend 5 minutes evaluating thhe experience
  • What communication technique worked?
  • What didn't?
